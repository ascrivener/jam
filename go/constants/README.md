# JAM Constants Generator

This tool generates constants for the JAM project by reading from external config files.

## How It Works

The constants generator:
1. Reads a config file from an external source
2. Maps the config values to internal constant names with appropriate types
3. Generates a `constants.go` file with all constants

## Usage

To generate the constants file:

```bash
cd go/constants
go run config_gen.go -config-path /path/to/external/config/file.go -network tiny
```

### Command Line Options

- `-config-path`: Path to the external config file (required)
- `-network`: Network name to use (optional, overrides the Network constant in the config file)
- `-output`: Output file path (default: constants.go)

## Example

If you have a config file from another project at `/path/to/external/tiny.go`:

```go
package configs

const (
    Network                     = "tiny"
    TotalValidators             = 6
    TotalCores                  = 2
    TicketEntriesPerValidator   = 3
    EpochLength                 = 12
    // ... other constants
)
```

You can generate your constants file with:

```bash
go run config_gen.go -config-path /path/to/external/tiny.go
```

This will create a `constants.go` file with properly mapped constants:

```go
// Code generated by config_gen.go; DO NOT EDIT.
// Generated from network: tiny
package constants

const NumValidators uint16 = 6 // V

const NumCores uint16 = 2 // C

const NumTicketEntries int = 3

// ... other constants
```

## Constant Mappings

The generator maps external config names to internal names with proper types:

| External Name | Internal Name | Type |
|---------------|---------------|------|
| TotalValidators | NumValidators | uint16 |
| TotalCores | NumCores | uint16 |
| TicketEntriesPerValidator | NumTicketEntries | int |
| EpochLength | NumTimeslotsPerEpoch | uint32 |
| ... | ... | ... |

Additional constants that aren't in the config file are also included in the generated output.
